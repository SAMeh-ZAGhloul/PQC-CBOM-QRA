## Root Directory Layout

```
cbom-platform/
├── .env.example                    # Template — copy to .env, never commit .env
├── .gitignore
├── .dockerignore
├── Makefile                        # All developer and operator commands
├── docker-compose.yml              # Full 19-service stack
├── docker-compose.override.yml     # Dev overrides: hot-reload, exposed ports
├── README.md                       # 5-step quickstart
│
├── secrets/                        # Docker secrets — NEVER commit contents
│   ├── .gitkeep
│   ├── jwt_private_key.pem         # Generated by gen-certs.sh
│   ├── jwt_public_key.pem          # Generated by gen-certs.sh
│   ├── db_password.txt
│   ├── redis_password.txt
│   ├── rabbitmq_password.txt
│   └── minio_password.txt
│
├── traefik/
│   ├── traefik.yml                 # Static config
│   ├── dynamic.yml                 # TLS options, middleware
│   └── certs/
│       ├── ca.crt                  # Self-signed CA (add to browser trust store)
│       ├── server.crt              # Wildcard server cert
│       └── server.key              # Server private key
│
├── scripts/
│   ├── gen-certs.sh                # Generate CA + wildcard TLS cert + JWT keys
│   ├── seed-db.sh                  # Create admin user + default groups
│   ├── backup.sh                   # pg_dump + upload to MinIO
│   ├── model-pull.sh               # ollama pull gemma2:2b
│   └── init-minio.sh               # Create buckets + enable versioning
│
├── db/
│   ├── init.sql                    # Full schema + RLS + seed RBAC roles
│   └── migrations/                 # Alembic migration files (auto-generated)
│
├── rabbitmq/
│   ├── rabbitmq.conf               # Queue definitions, DLQ policy
│   └── definitions.json            # Pre-declared exchanges/queues
│
├── zeek/
│   ├── local.zeek                  # Site policy: load SSL/SSH/hash scripts
│   └── scripts/
│       └── crypto-detection.zeek   # Cipher suite → algorithm mapping
│
├── api/                            # FastAPI backend
├── frontend/                       # React + Vite SPA
├── orchestrator/                   # Discovery orchestrator
├── scanners/                       # Celery scanner workers (all types)
├── magika-service/                 # Magika HTTP microservice
├── cbom-generator/                 # CycloneDX CBOM assembler
├── scoring-engine/                 # QARS + QSRI engines
├── traffic-sim/                    # Locust + sample apps
│
└── shared/                         # Host-mounted shared directories
    └── zeek-logs/                  # Zeek JSON log output (bind mount)
```

## Per-Service Directory Pattern

Every Python service follows this exact layout:

```
service-name/
├── Dockerfile
├── pyproject.toml
├── src/
│   └── service_name/           # snake_case module name
│       ├── __init__.py
│       ├── main.py             # entry point / app factory
│       ├── config.py           # Settings (pydantic-settings)
│       ├── models/             # Pydantic + SQLAlchemy models
│       ├── services/           # Business logic
│       └── utils/              # Helpers
└── tests/
    ├── conftest.py
    ├── unit/
    └── integration/
```

## Dockerfile Template (Python services)

```dockerfile
FROM python:3.12-slim

# Security: run as non-root
RUN groupadd -r cbom && useradd -r -g cbom cbom

WORKDIR /app

# Install system deps (customize per service)
RUN apt-get update && apt-get install -y --no-install-recommends \
    curl \
    && rm -rf /var/lib/apt/lists/*

# Install Python deps
COPY pyproject.toml .
RUN pip install --no-cache-dir -e .

# Copy source
COPY src/ ./src/

USER cbom

# Health check (customize port per service)
HEALTHCHECK --interval=30s --timeout=10s --start-period=30s --retries=3 \
  CMD curl -f http://localhost:8000/health || exit 1

CMD ["uvicorn", "service_name.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

## README.md (5-Step Quickstart)

```markdown
# Quantum-Safe CBOM Discovery Platform

## Prerequisites
- Docker 26+ and Docker Compose v2
- Linux or macOS host
- 16 GB RAM minimum (32 GB recommended)
- 8 CPU cores minimum

## Quickstart

### Step 1 — Clone and configure
git clone <repo-url> cbom-platform && cd cbom-platform
cp .env.example .env
# Edit .env to set your network interface for Zeek (ZEEK_INTERFACE=eth0)

### Step 2 — Generate certificates and secrets
make setup

### Step 3 — Pull the SLM model (one-time, ~3 GB download)
make pull-model

### Step 4 — Start the platform
make up

### Step 5 — Open the dashboard
https://localhost
# Accept the self-signed certificate warning
# Login: admin / (password set during make setup)

## Useful commands
make logs          # tail all container logs
make status        # show container health
make backup        # manual CBOM backup to MinIO
make down          # stop all containers
make reset         # stop + delete all volumes (DESTRUCTIVE)
```
